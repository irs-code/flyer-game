# flyer-game

## Project Overview
The flyer-game is a mini project developed as part of Code.org Lesson 24. This game allows players to navigate a character through obstacles in a fun and engaging way, focusing on enhancing programming skills.

## How to Run

### Code.org App Lab
You can also import this project directly into Code.org's App Lab for further development and modification. Simply create a new project in App Lab and copy the code files into the respective sections.

## Controls/Gameplay
- **Arrow Keys**: Move the character up, down, left, or right.
The objective is to avoid obstacles and achieve the highest score possible.

## Customization Ideas
- Add new levels or game modes to enhance gameplay.

## Assets / Sprites

To run or remake this project, you’ll need these sprites/assets available in your Code.org project’s **Animation Tab** (or in an `assets/` folder if you export elsewhere).

### Required sprites (checklist)

**Player**
- [ ] `player_flyer` — the main character/flyer (idle or single frame is fine)

**Obstacles / Hazards**
- [ ] `obstacle_1` — first obstacle (ex: cone, rock, spike)
- [ ] `obstacle_2` — second obstacle (optional variety)

**Collectibles / Score**
- [ ] `target` — item to pick up for points (ex: coin, star)

### Asset guidelines

- **Naming:** use the exact names above (or update the code to match your names).
- **Sizing:** keep sprites roughly consistent (e.g., player and obstacles scaled so collisions feel fair).

If you use different names, update the code anywhere it calls `setAnimation(...)` to match.
